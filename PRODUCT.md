# Product

## Register

product

## Users

The primary user is a technical individual running a private local or trusted LAN tool for comparing Alibaba Cloud Token Plan models. They are usually asking questions, comparing model output quality, continuing per-model conversations, generating images, and optionally letting the local project Agent read relevant files.

## Product Purpose

Token Plan Arena is a personal multi-model workbench. It lets one prompt fan out to selected models, keeps each model's memory independent, shows streaming answers side by side, stores history in Postgres, and keeps API keys on the server. Success means the user can quickly choose models, ask once, compare clearly, and continue with one model without losing context or mixing other models' replies.

## Brand Personality

Calm, precise, capable. The interface should feel like a serious local command center for model work: dense enough for repeated use, polished enough to feel intentional, and quiet enough that the answers remain the focus.

## Anti-references

Do not make it a marketing page. Avoid flashy gradients, game-like styling, decorative sci-fi excess, purple-blue AI default palettes, and big hero layouts. Avoid over-rounded toy-like cards, bloated vertical spacing, and decorative motion that does not communicate state.

## Design Principles

- Make comparison effortless: selected models, current run state, and each answer's context should be scannable at a glance.
- Keep controls close to the task: project Agent, images, model filters, and history should feel like a compact workbench rather than separate pages.
- Preserve trust: show configuration, rate-limit, context-reading, and error states clearly without leaking secrets.
- Use restraint with presence: the UI can feel sharp and atmospheric, but visual effects must support hierarchy and focus.
- Design for repeated sessions: compact density, predictable controls, and clear per-model memory matter more than novelty.

## Accessibility & Inclusion

Target WCAG AA contrast for text and controls. Support reduced motion. Do not rely on color alone for state; pair color with labels, icons, or text. Keep keyboard-focus states visible and maintain readable Chinese UI copy.
